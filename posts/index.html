마크다운 문법

1. 현재 위치 확인

현재 디렉터리 확인

  • 현재 내가 어느 위치에 있는지 절대 경로로 출력
$ pwd
C:/Users/kimna/Desktop/test

바탕화면으로 이동

  • cd 명령어로 원하는 디렉터리로 이동
$ cd ~/Desktop

현재 위치의 파일 폴더 목록 보기

  • 현재 디렉터리 안의 파일과 폴더를 나열
$ ls
  • 숨김 파일 포함, 상세 정보까지 보기
$ ls -al

2. 폴더(디렉터리) 만들기 및 이동

test 폴더 안에서 새로운 폴더 만들기

$ mkdir linux_practice
  • 중첩 폴더를 한번에 만들기 (-p 옵션)
$ mkdir -p linux_practice/src/utils

만든 폴더로 이동

$ cd linux_practice

상위 폴더로 이동하기

$ cd ..

3. 파일 만들기 및 내용 확인

빈 파일 만들기 (tuoch)

  • touch로 빈 파일 생성
$ cd ~개인경로/linux_practice
$ touch hello.txt
  • >는 덮어쓰기, >>는 내용 추가
    • 가급적이면 >는 사용 금지
$ echo "Hello, Linux!" > hello.txt
$ echo "두 번째 줄 추가" >> hello.txt

파일 내용 확인

  • 파일 전체 내용을 터미널에 출력
$ cat hello.txt

vi 편집기 익히기

$ vi memo.txt
  • 기본 문법
  • vi 편집기 : 개발자 필수 / 분석가 옵션 / 기획자, 공부하지 마세요!
    • 개발자의 경우 : 앞으로 markdown 작업할 때, vi 편집기로 하는 것 추천
키동작
i삽입 모드 진입 (현재 커서 앞)
ESC일반 모드로 복귀
:w저장
:q종료
:wq저장 후 종료
:q!저장 없이 강제 종료

5. 파일 폴더 복사, 이동, 삭제

파일 복사 (cp)

  • cp 원본 복사본 형식
$ cp hello.txt hello_backup.txt
  • 폴더 전체 복사 (-r 옵션)
$ cp -r linux_practice/ linux_practice_backup/

파일 폴더 이동 / 이름 변경 (mv)

  • 같은 위치에서 이름만 바꾸면 이름 변경
$ mv hello_backup.txt hello_copy.txt
  • archive 폴더 생성
  • hello_copy.txt 파일을 archive 이동 (mv)
$ mkdir archive
$ mv hello_copy.txt archive/

파일 삭제 (rm)

  • 파일 삭제
$ rm 파일명
  • 폴더 전체 삭제 (-r 옵션)
$ rm - r archive
  • 강제 삭제 -rf
$ rm -rf 폴더/파일/...

Git과 Hugo를 받고 Github로 블로그 만들기

우선 Git을 다운로드 받아야 하는데 공식 홈페이지 https://git-scm.com/ 에서 OS에 맞게 다운로드를 하여 설치한다.

그 후 윈도우 기준으로 PowerShell에서 명령어를 통해 hugo 다운로드 winget install Hugo.Hugo.Extended

다운로드가 완료 되면 PowerShell을 닫았다가 재실행 하고 hugo version 으로 설치가 되었는지 확인을 해본다.

Github를 이용하여 블로그를 만들 예정이므로 https://github.com/ 에서 깃허브에 가입한다.

Repositories 옆의 New를 눌러 새로운 repo를 생성하는데 repo 이름은 사용자이름.github.io 로 하여 public으로 생성하면 해당 주소로 된 블로그를 만들 수 있다.
